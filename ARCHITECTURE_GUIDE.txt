===========================================================
WAREHOUSE MANAGEMENT - CLEAN ARCHITECTURE GUIDE (.NET 8)
===========================================================

Proyek ini menggunakan Clean Architecture (Onion Architecture).
Prinsip utamanya adalah: "Dependencies Flow Inwards".
Artinya, bagian dalam (Bisnis) tidak boleh tahu apa yang terjadi 
di bagian luar (Database/UI).

-----------------------------------------------------------
1. DOMAIN LAYER (The Heart)
-----------------------------------------------------------
- Ini adalah lapisan paling dalam dan paling suci.
- Tidak boleh memiliki ketergantungan (dependency) ke library luar.
- Isinya:
  * Entities: Representasi tabel database (Product, Stock, dll).
  * Enums: Pilihan tetap (Inbound, Outbound).
  * Exceptions: Error khusus untuk logika bisnis.
- Tujuan: Jika kita ganti database dari SQL Server ke MongoDB, 
  layer ini TIDAK AKAN BERUBAH.

-----------------------------------------------------------
2. APPLICATION LAYER (The Brain)
-----------------------------------------------------------
- Berisi logika bisnis (Business Logic) dan Use Cases.
- Bergantung HANYA pada Domain Layer.
- Isinya:
  * Interfaces: Kontrak untuk apa yang bisa dilakukan (IRepository).
  * DTOs (Data Transfer Objects): Objek untuk kirim data ke API.
  * Mappers: Mengubah Entity menjadi DTO dan sebaliknya.
  * Logika Konversi: Tempat kita menghitung Box ke Pcs.

-----------------------------------------------------------
3. INFRASTRUCTURE LAYER (The Tools)
-----------------------------------------------------------
- Berisi implementasi teknis yang berhubungan dengan luar sistem.
- Bergantung pada Application Layer.
- Isinya:
  * Data: DbContext (Entity Framework Core).
  * Repositories: Cara nyata mengambil data dari SQL/SQLite.
  * External Services: Kirim email, akses file system, dll.

-----------------------------------------------------------
4. WebAPI LAYER (The Door)
-----------------------------------------------------------
- Pintu masuk utama aplikasi.
- Bertanggung jawab menerima request HTTP dan mengembalikan JSON.
- Isinya:
  * Controllers: Menghubungkan input user ke Application Layer.
  * Program.cs: Tempat setting "Dependency Injection".
  * appsettings.json: Tempat menyimpan Connection String.

-----------------------------------------------------------
ALUR DATA (Data Flow):
User Request -> WebAPI -> Application -> Domain -> Infrastructure
-----------------------------------------------------------
